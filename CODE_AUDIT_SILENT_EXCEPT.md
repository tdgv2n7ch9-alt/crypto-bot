# Аудит: `except: pass` / `except Exception: pass` (READ-ONLY, владелец, 2026-07-22)

Фоновая очередь, приоритет "аудит кода read-only" (CLAUDE.md, вечный
бэклог п.7): поиск известного класса дефекта (`except` без `log.error`) --
список файл:строка, **без правок**. Метод: построчный скан всех `.py`
файлов репозитория (кроме `tests/`, воркдеревьев, бэкапов) на паттерн

мягкий класс, здесь не собирался).

### Почему это безопасно применить (оценка риска, не решение)

Оба патча -- ТОЛЬКО добавляют логирование в уже существующий `except`,
НЕ меняют условия/поток исполнения/боевую логику сигналов, НЕ трогают
чек-лист/гейты. Класс изменения -- ровно тот же, что уже был применён и
проверен для `run_watchdog()` 2026-07-19 (та же формула:
`except Exception as e: log.error(...)`). По классификации CLAUDE.md
("Автономия v2") это выглядит как класс B (известный паттерн, тот же
класс бага, что уже решался и проверялся раньше) -- но затрагивает
`bot.py` (боевой код) и обозначен владельцем явно как "не применять,
это gate владельца", поэтому остаётся ТОЛЬКО предложением до отдельного
"да".

**Ничего не применено. Live-логика/гейты/чек-лист не менялись. `PAUSE_
LIVE_SIGNAL_EMISSION` без изменений (`true`), промоушены заблокированы.**
