# Дизайн-заметка: DefiLlama API — точечное бесплатное дополнение (Фаза C + резерв к CoinGecko)

Владелец, 2026-07-21: DefiLlama уже в роадмапе (Фаза C, TVL-слой) —
разведка на предмет ЧТО ИМЕННО бесплатно и есть ли реальное применение
ПРЯМО СЕЙЧАС (резерв к CoinGecko), не только копилка на будущее. Только
заметка, код не подключал.

## Бесплатно (совпадает с методологией)

### 1. Стейблкоины — `/stablecoins`, `/stablecoincharts/{chain}`, `/stablecoinchains`

[проверено — https://api-docs.defillama.com/, endpoint-таблица]. Прямое
попадание в уже используемую логику "USDT mcap = новая ликвидность" +
"стейблы 30д" (уже в брифе, по словам владельца) — резерв/апгрейд
источника, не новая идея.

### 2. Цены — `/prices/current/{coins}`, `/prices/historical/{timestamp}/{coins}`, `/chart/{coins}`

[проверено — https://api-docs.defillama.com/, base URL `https://api.llama.fi`].
**Важная честная оговорка, ПРЯМО влияющая на предложенное ДЕЙСТВИЕ ниже**:
по данным `docs.llama.fi` (через агрегированный поиск, идентификаторы
монет используют формат `coingecko:<id>`, например `coingecko:bitcoin`) —
DefiLlama **не независимый ценовой источник для большинства токенов**,
а прогоняет цену ЧЕРЕЗ CoinGecko API (для токенов, у которых нет
собственного on-chain расчёта) [вторичный источник, поиск по
"coins.llama.fi API Schema" -- не смог получить первоисточник
docs.llama.fi/coin-prices-api напрямую, страница отдаёт 404 через fetch].

**Что это значит для "резерва к CoinGecko"**: DefiLlama НЕ спасёт от
полного простоя CoinGecko (тот же апстрим). НО у DefiLlama, вероятно, СВОЙ
кэш/буфер поверх CoinGecko (типичная архитектура агрегаторов) — то есть
во время КОРОТКОГО circuit breaker-окна (наша реальная боль -- 30-60с
паузы, не многочасовой полный даунтайм CoinGecko) DefiLlama вполне может
отдать чуть устаревшую, но ДОСТУПНУЮ цену, пока наш прямой запрос к
CoinGecko блокирован собственным circuit breaker'ом. Это НЕ независимая
проверка, а буфер сглаживания коротких пауз -- честно, не то же самое, что
"второй независимый источник данных", но всё равно может снизить видимые
паузы в бриф-отчётах. Рекомендация -- не отменять текущий circuit breaker
у CoinGecko, а добавить DefiLlama как fallback ВНУТРИ уже существующего
`_cg_get()`-паттерна (при активном cooldown -- пробовать DefiLlama прежде,
чем отдавать "протухший кэш").

### 3. TVL — `/v2/historicalChainTvl`, `/v2/chains`

[проверено — https://api-docs.defillama.com/]. TVL-слой Фазы C, как и
задумано в роадмапе -- ничего нового к этому пункту, просто подтверждение,
что оба эндпоинта бесплатны.

### Свободный тир — лимиты

Официальная документация НЕ публикует точное число (только "Standard" без
цифры) [проверено — https://api-docs.defillama.com/, не даёт число].
Вторичный источник (community skill-doc, НЕ первоисточник) называет
"~500 запросов/5 мин (≈100/мин)" для free и "1000 req/мин, 1M вызовов/мес"
для платного API-тира [не первоисточник, не выдаю за подтверждённое].
Официально подтверждено только: без авторизации, без явного строгого
порога в первичных доках.

## Платное — ДВА РАЗНЫХ продукта, не путать (важная находка)

**Честная находка при разведке**: DefiLlama продаёт ДВЕ разные подписки
под похожими названиями:

1. **"Pro" (дашборд-подписка) — $49/мес** [проверено —
   https://docs.llama.fi/pro-api]: LlamaAI (5 вопросов/день), кастомные
   дашборды, CSV-экспорт, премиум-лента. **API ДОСТУП ЯВНО НЕ ВКЛЮЧЁН** --
   дословно на странице: "API access is not included in the Pro plan".
2. **API-тир — $300/мес** (то, что владелец и имел в виду под "API Plan"):
   выше лимиты + доп. эндпоинты. Точный список доп. эндпоинтов первоисточником
   подтвердить не смог (страница `docs.llama.fi/pro-api.md`/GitHub-путь
   отдаёт 404 через доступные мне инструменты) — упомянутые владельцем
   inflows/outflows, chainAssets, aggregate metrics, perps/lsd yields,
   volumes, stablecoindominance, MCP server правдоподобны (частично
   встречаются в сторонних описаниях -- "token inflows/outflows between
   timestamps", "getChainAssets()"), но НЕ проверены мной первоисточником
   в этой заметке -- если у владельца есть прямой скриншот/страница,
   доверять ей больше, чем этой заметке.

## Оценка для нашей методологии (согласен с владельцем)

Слабее Arkham/Coinglass для НАШЕГО профиля (оператор/деривативы/онхейн-
выгрузки, не DeFi/TVL-мир) — DefiLlama сильна там, где мы не работаем
(протокольный TVL, DeFi-yield). Берём точечно бесплатное (стейблы + цены-
резерв + TVL для Фазы C), не весь фид.

## ДЕЙСТВИЕ (после диагноза WR, кандидат на раннюю интеграцию)

`/prices/current/{coins}` как fallback внутри `bot._cg_get()`/
`get_all_coins()` при активном CoinGecko circuit breaker (`_cg_cooldown_until`) --
вместо немедленной отдачи "протухшего кэша", один дополнительный бесплатный
HTTP-вызов к `api.llama.fi` перед тем, как сдаться. Снижает видимые паузы
в стартовых/утренних отчётах (текущая боль, circuit breaker виден в каждом
интеграционном отчёте бота). Честно: НЕ гарантированно спасает от 429,
если DefiLlama сама в моменте бьёт по тому же CoinGecko апстриму для
конкретного токена (см. оговорку выше) — но кэш-буфер агрегатора обычно
сглаживает именно короткие паузы, которые и есть наша реальная проблема.

Код НЕ писал -- это дизайн-предложение для следующего пакета после
диагноза WR, реализация (если владелец даст "да") -- отдельная, небольшая
по объёму задача (один доп. HTTP-путь в уже существующей функции).
