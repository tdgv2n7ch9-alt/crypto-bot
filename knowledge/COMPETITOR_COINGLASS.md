# Дизайн-заметка: Coinglass как референс для деривативного дашборда + Волны 1

Владелец, 2026-07-21: Coinglass — визуальный образец готовой Волны 1
(#288 мега-пакет). Разбор ПОДАЧИ + состава метрик (в отличие от заметки по
CMC — здесь состав тоже релевантен, не только форма, поскольку Coinglass
специализируется на деривативах, той же области, что Волна 1). Приоритет:
при старте дизайна Волны 1, после диагноза WR. Код не трогал.

## Что берём (совпадает с методологией проекта)

### 1. Топ-виджеты OI / Liquidation / Avg RSI / Alt Season в шапку Mini App

OI и ликвидации у нас частично уже есть (см. таблицу ниже) — собрать в
единую ленту виджетов по образцу Coinglass, не изобретать заново. Avg
RSI/Alt Season — та же честная оговорка, что в заметке по CMC
(`miniapp_radar_cmc_pattern_note.md`): нет готовой агрегированной метрики
в проекте, нужна отдельная проверка/реализация, не выдумывать.

### 2. Long/Short по СЧЕТАМ топ-трейдеров (Top Trader L/S)

**Прямое совпадение с методологией** — "толпа в шорте = топливо сквиза"
(кейс AKE, `METHODOLOGY_CORE.md`). **Кандидат Волны 1.** Честная оговорка
по текущему состоянию: `bot._get_ls_ratio()` уже существует и живой, но
использует Bybit `v5/market/account-ratio` — это **ВСЕ аккаунты**, не
конкретно топ-трейдеры (крупные позиции). Bybit отдельно предоставляет
`v5/market/account-ratio` ТОЛЬКО в этом виде на публичном API (без
авторизации) — честно не проверял сегодня первоисточником, есть ли у
Bybit вообще public top-trader-specific endpoint, отдельный от этого,
или "топ-трейдер" срез — платная привилегия у большинства бирж (в т.ч.
похоже и у Coinglass — агрегация топ-трейдеров с разных бирж). Требует
отдельной проверки перед тем, как закладывать в дизайн Волны 1 как
"бесплатно доступно".

### 3. OI Change % по окнам 5m/30m/4h

Динамика поверх нашей статичной OI-матрицы (`fa_engine._oi_matrix()`).
**Волна 1.** Текущее состояние: `bot._get_oi_change()` считает изменение
OI только между последовательными опросами процесса (in-memory
`_OI_HISTORY`, не multi-window 5m/30m/4h одновременно, и обнуляется при
каждом рестарте — не персистентная история). Многооконная версия —
реальная новая работа Волны 1, не просто "включить флаг".

### 4. Funding Rate тепловой заливкой

Подача для карточки (не новый источник данных — `bot.get_funding_rate()`/
`_get_funding_pct()` уже живые). Чисто визуальный слой, дешёвый в
реализации относительно остальных пунктов.

### 5. Bitcoin Exchange Balance (отток с бирж)

Смежно с netflow CEX (`onchain_watch.CEX_DEPOSIT_ADDRESSES` — механизм
существует, но САМ модуль `onchain_watch.py` архивирован сегодня вместе с
закрытием кейса BANK, см. `archive/README.md`). **Волна 1** явно включает
`unlock_on_cex_balance`/netflow как отдельный пункт (приоритет повышен
владельцем сегодня, см. `NEXT_PACKAGE.md`) — переиспользование архивного
механизма, не новая идея, но нужно вынести из архива/обобщить под
произвольный символ (сейчас адреса были специфичны под BANK/Lorenzo
Protocol).

## Явно НЕ брать (владелец, подтверждено)

- Stocks/Gold/SP500 в одной таблице с крипто-деривативами — вне скоупа
  продукта (макро-контекст уже есть в ленте виджетов отдельно, смешивать
  в ОДНУ таблицу с деривативами не нужно).
- Top Gainers — мемы без методологии, осознанно избегаем (тот же принцип,
  что уже действует в проекте: сетапы без структурного обоснования не
  подаём как сигнал).
- Heatmap как ОСНОВА экрана — Coinglass строит вокруг heatmap как
  центрального элемента (широта охвата); наш продукт — глубина по
  конкретным PD-Score кандидатам, не широкий обзорный дашборд. Взять
  ТОЛЬКО формат/визуальный язык heatmap (тепловая заливка funding, п.4
  выше), не саму архитектуру "весь рынок одним экраном".
