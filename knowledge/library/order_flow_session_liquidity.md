# Order Flow — "здоровое" vs "нездоровое" движение цены между сессиями

**Новая тема** — не пересекается напрямую ни с одним разделом
METHODOLOGY_CORE.md (проверено: термин "order flow"/"здоровое движение"
там не встречается). Спикер сам признаёт, что название условно ("order
flow", "healthy movement", "манки бизнес" — синонимы одного и того же
наблюдения).

**Ключевые тезисы**:
1. **Правило переноса тренда между сессиями**: если сессия (например,
   Азия) шла трендово И по ходу СНИМАЛА собственную оставленную
   ликвидность (не копила её) — следующая сессия (Лондон) *вероятнее*
   продолжит то же направление ("здоровое движение" = continuation).
2. **Обратное правило**: если сессия шла трендово, но ОСТАВИЛА
   непроверенную ликвидность позади (невыбитые хаи/лои) — следующая
   сессия вероятнее развернётся, чтобы сначала собрать эту оставленную
   ликвидность, прежде чем продолжать.
3. **Первые 15-30 минут открытия сессии — ключевое окно** (для Франкфурта
   спикер называет именно первые ~15 минут): первый свип/манипуляция в
   этом окне считается значимым сигналом направления.
4. **Локальное правило BOS внутри "order flow"-контекста**: пока цена
   идёт "здоровым" направленным движением, мелкие хаи/лои НЕ считаются
   сломом структуры — это "индукция" (ловушка) перед настоящей зоной
   интереса. Настоящий BOS для этой логики — только пробой И закрепление
   ЗА уровнем, который РАНЕЕ снял ликвидность (не любой мелкий экстремум).
5. Правило работает не только для пары Азия→Лондон — применимо к любой
   паре последовательных сессий/killzone (сам спикер уточняет это в ответ
   на вопрос слушателя).

**Честно н/д**: точные пороги "сколько ликвидности считается
'оставленной'" не даны — оценка на глаз по графику, не формализована в
это видео.

**Точки соприкосновения с ботом**: движок НЕ реализует межсессионную
логику "снята/не снята ликвидность прошлой сессии → предсказание
continuation/reversal следующей" — честный пробел, не оценивался ранее
(killzone-логика в §8 METHODOLOGY_CORE.md касается только времени сессий,
не переноса ликвидности между ними).
