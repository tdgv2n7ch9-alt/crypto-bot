# Инструктор E, Урок 3: Открытия, контекст открытий (~108 мин, прочитано выборочно)

**Находка 1 — два конкурирующих определения "дневного открытия", источник
явно выбирает НЕ тот, что уже задокументирован**: уже известный gap
`KNOWLEDGE_GAPS.md` п.23 говорит про NY Midnight (00:00 NY) как "истинное"
(True Daily Open) дневное открытие (источник — `39. ММХМ.pdf`, ICT-школа).
**Этот источник явно предпочитает ДРУГОЕ** — TradingView/GMT-based Daily
Open (свеча открытия по GMT, НЕ по NY Midnight): "я использую только
TradingView, True Daily Open не использую... этот уровень больше имеет
значение для крипты, True Daily Open используют для форекса". То есть
для крипты (наш случай) этот источник рекомендует ИМЕННО GMT-открытие, а
не NY Midnight — прямое, конкретное дополнение/частичное противоречие
уже известному пункту #23, честно зафиксировано как несогласие между
источниками, не решаю сам, какой "правильный".

**Daily/Weekly Open как ось bias-навигации**: цена выше открытия = бычий
bias актуален, ниже = медвежий — вход "на уровне открытия или под ним"
(для лонга) ловит остаток тела/тени дневной свечи. Если цена уходит
"не в ту сторону" от открытия — bias под вопросом, ищем зону манипуляции.

**Находка 2 — Previous Daily/Weekly/Monthly High/Low как ЦЕЛИ (targets), не
только зоны входа**: подтверждает и РАСШИРЯЕТ уже найденный `dwm_extremes_
strategy.md` (`KNOWLEDGE_GAPS.md` п.32) — там угол был "вход через false
breakout от DWM-уровня", здесь — тот же класс уровней (Previous Weekly
High/Low, Previous Daily Low, Previous Monthly Low) используется как
ЦЕЛЬ движения (TP), не как сигнал входа. Тот же неимплементированный
класс данных (см. п.32), просто другое применение.

**Модуль бота**: подтверждает существующий пробел #32 (DWM extremes) +
уточняет/оспаривает #23 (NY Midnight vs GMT daily open) — обе находки уже
покрыты существующими пунктами, добавляю деталь, не новый номер.
