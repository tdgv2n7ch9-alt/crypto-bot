# 26. Стрим AMD/Wyckoff Часть 1 (поток 7.0) — стандартная схема Wyckoff, подтверждение известного

**Часть 1 из пары со `знакомым файлом` "27. Стрим Wyckoff Часть 2"** (уже
конспектирован в прошлой сессии, `amd_po3_wyckoff_day_templates.md`, который
нашёл ключевой различающий признак Reaccumulation vs Distribution — с какой
стороны рэнджа снимается ликвидность). Эта, первая часть (102 мин, прочитано
~40%) — фундаментальная, стандартная схема Wyckoff Accumulation Type 1/2
(Selling Climax → AR → ST → Spring/STB) — классическая, широко известная
методология, не проприетарная этому инструктору, содержательно совпадает с
уже задокументированным в `METHODOLOGY_CORE.md` §18.2-18.6/§6.

**Явно подтверждает уже известное, не новое**:
- Spring/STB (тест уровня перед разворотом) может быть либо чистой тенью,
  либо "спонсированной свечой" (manipulation-свеча + поглощающая свеча) — тот
  же паттерн, что уже задокументирован как `KNOWLEDGE_GAPS.md` п.27
  ("Sponsored Candle").
- Явно сказано "смотрю и по свипам (тень), и по телу" — то есть для входа по
  Spring этот инструктор НЕ требует строго body-close (в отличие от строгой
  позиции по обычному BOS у других источников, см. батч 11) — довольно
  гибкая, не догматичная позиция конкретно для Wyckoff-контекста.
- Заявленный винрейт ">50%" — субъективная оценка инструктора без
  бэктест-цифр, не переносится как факт.

**Модуль бота**: те же, что уже перечислены в `amd_po3_wyckoff_day_templates.md`
(`bot.py:7443-7484` `amd_phase`/`amd_label` — считается, но не используется
дальше по коду, уже задокументированный мёртвый код, не добавляю дубль).
