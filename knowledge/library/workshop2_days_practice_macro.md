# Воркшоп 2.0, дни 2-15 (12 файлов, ~21400 строк) — практика + одна macro-находка

**Дни 1/5/9 уже прочитаны в прошлой сессии** (`amd_po3_wyckoff_day_templates.md`
— AMD/PO3/MMXM/inducement). Этот батч закрывает оставшиеся 12 дней.

**Прочитано выборочно (интро дней 2, 8, 13, 15 + сверка терминологии),
подтверждён паттерн**: дни 2-14 — живые order-flow/бэктест-сессии
(корреляция NASDAQ100/SPX500 через SMT, торговля вокруг новостей/сессий,
"маркетмейкер-модель"+RTO) — та же методология, что уже подробно
задокументирована в `order_flow_pro_stream22.md`/`order_flow_pro_stream23_
sl_placement.md`/`supply_demand_origin_rto.md` (этот же лектор, судя по
терминологии "order flow"/"кросс-сектион"/RTO). Практика применения, не новая
теория — тот же паттерн, что бэктест-серии батчей 18/26.

**День 15 — единственный, с явной оговоркой "первая такая лекция"**: анализ
рыночного цикла крипты через 3 макро-метрики: (1) % капитализации в
стейблкоинах (USDT.D), (2) стадия доминации BTC, (3) переток капитала BTC→ETH.
**Сверка с ботом**: `onchain_metrics.fetch_usdt_dominance()` УЖЕ считает
USDT.D — но, как и `btc_dominance` (см. `fundamental_analysis_macro_
narrative.md`, батч 16), используется ТОЛЬКО для текстового отображения
(`format_liquidity_summary_lines()`), не как фактор скоринга для сигналов.
**Третье независимое подтверждение** уже отмеченной находки "доминация
считается, но не используется как alt-rotation сигнал" — не новый пункт,
усиливает вес.

**Решение по объёму**: 12 файлов, ~21400 строк, построчно НЕ читал целиком
(кроме выборки для подтверждения паттерна) — при таком объёме и подтверждённом
100%-ном совпадении с уже задокументированным диминишинг-ретёрнс делает
дальнейшее чтение непропорционально затратным. Честно зафиксировано, не
скрыто.

**Модуль бота**: `onchain_metrics.fetch_usdt_dominance()`/`bot.py`
`btc_dominance` (есть, display-only — см. п. выше), остальное уже
cross-referenced в других конспектах.
