# Вводный блок курса (поток 7.0, Beginner): платформа обучения, TradingView, работа с биржей

**Честно: платформенный онбординг, не торговая методология.** Все три файла — вводные
уроки нового потока курса (регистрация на учебной платформе, интерфейс TradingView,
регистрация/верификация/работа с биржей Binance). Ни один не содержит авторской SMC/ICT-
методологии этого курса — она начинается в следующих уроках (см. остальные конспекты
батча #205). Честно фиксирую отсутствие содержания, а не выдумываю разбор ради объёма.

**Урок 1. Вводный урок** — целиком логистика учебной платформы (логин/пароль, привязка
Telegram к боту-ассистенту, доступ к учебному чату, структура модулей "видео → методический
материал → тест → домашнее задание", вкладка статистики). Ноль торгового содержания.

**Урок 2. Работа с TradingView** — интерфейс платформы: торговые пары, таймфреймы (счётчик
свечей: 1д = 6×4ч = 96×15м), японские свечи выбраны намеренно ("учитываем манипуляции
тени" — линейный график их не показывает), индикаторы/избранное, оповещения по цене,
симулятор рынка для практики на историческом графике, инструменты рисования (линия тренда,
уровень Фибоначчи, прямоугольник для зон интереса, длинная/короткая позиция для расчёта
риска). Всё — чистый UI-тур, ни одного конкретного торгового правила.

**Урок 3. Работа с биржей** — единственный из трёх с содержанием чуть шире чистого UI:
- Регистрация/верификация (3 уровня KYC), 2FA (Google Authenticator + телефон + email).
- Спотовая торговля: лимитный/маркет/стоп-лимит ордера — лимитный ждёт цену бессрочно,
  маркет исполняется по лучшей доступной цене стакана (явно назван невыгодным — "маркет
  ордера в основном пользуется очень редко, в основном всё работает через лимитные").
- Фьючерсы: **funding rate** явно назван и объяснён (ставка каждые 8 часов, лонги платят
  шортам или наоборот, механизм балансировки открытого интереса) — бот уже использует
  funding через `bot.get_funding_rate()`/`_get_funding_rates()` (chek-лист `fa_engine.py`,
  patch09_oi shadow-контур) — соответствие подтверждено, курс не добавляет ничего сверх
  уже реализованного.
- **Изолированная vs кросс-маржа**: изолированная — обеспечение только на конкретную
  сделку (max loss = выделенная маржа), кросс — под угрозой весь баланс фьючерсного
  кошелька. Явная рекомендация для новичков — изолированная маржа.
- **TP/SL trigger: "последняя цена" vs "маркировка" (mark price)** — единственная деталь
  в этих 3 уроках с реальным техническим нюансом: "последняя цена" срабатывает по касанию
  тенью, "маркировка" требует, чтобы СВЕЧА ЗАКРЫЛАСЬ телом в диапазоне цены триггера
  (пример в уроке: тень дошла до 30000 и откатилась — по last price закрыло бы сделку, по
  mark price — нет, раз тело закрылось ниже). Это тот же класс различия wick-vs-body, что
  бот уже применяет в других местах (`order_block_live`/`order_block_methodology` — body
  vs wick геометрия order block, "Патч 07"; `bos_body_close_shadow` поле в shadow-записях)
  — сам бот НЕ формулирует эту пару принципов как выбор TP/SL-триггера явно (это
  биржевая механика исполнения ордера, не часть внутреннего анализа), поэтому это не gap
  бота, а полезный концептуальный мостик: тот же принцип "тело важнее тени", который бот
  применяет к SMC-паттернам, биржа применяет к исполнению TP/SL через mark price.

**Модуль бота:** funding — `bot.get_funding_rate()`/`_get_funding_rates()` (уже
реализовано, курс ничего нового не добавляет). Wick-vs-body принцип TP/SL-триггера —
нет прямого модуля (это подсказка ПОЛЬЗОВАТЕЛЮ бота при ручном выставлении TP/SL на
бирже, не внутренняя логика анализа) — не gap, не относится к тому, что бот считает сам.
