# 42. Стрим Продвинутый Price Action (поток 7.0) — честно НЕ канон, авторская насмотренность без методички

**Важная оговорка САМОГО источника (протокол правды требует это выделить)**:
инструктор явно говорит, что для этого модуля НЕТ методички — "тема впервые,
основывается на нашем опыте, насмотренности", и прямо предупреждает: "прошу
вас с головой, только через бэктесты" — эти свечные формации часто
встречаются на графике САМИ ПО СЕБЕ, и без остальной методологии (структура/
ликвидность/зоны) "можно ломать дров". То есть это НЕ проверенный канон, а
субъективная эвристика конкретных инструкторов, представленная как таковая.

**Три конкретных свечных формации-подтверждения** (используются как ранний
вход в зоне интереса ИЛИ доп-подтверждение слома, НЕ как самостоятельный
сетап — сам источник это подчёркивает):
1. **Поглощение (Engulfing)** — тело одной свечи полностью закрывает тело
   предыдущей. Концептуально уже покрыто ботом через order block (`find_sr_
   zones`/поглощающая свеча = основа определения OB, см. `KNOWLEDGE_GAPS.md`
   п.6), не новый примитив.
2. **Рельсы (Rails)** — 2+ свечи с плотным телом и почти отсутствующими
   тенями подряд — сигнал сильного импульса/подтверждения направления. Это
   ГЕНУИННО отдельный от order block примитив (order block — ОДНА
   поглощающая свеча, рельсы — ПАРА/серия плотнотелых свечей без явного
   поглощения) — не покрыт ни gap п.6 (Order Block), ни п.26 (крупные
   мультибаровые паттерны вроде голова-плечи/флаги/треугольники, Rails —
   мельче их всех). Кандидат в отдельный, узкий пункт GAPS, если владелец
   захочет — честно НЕ добавляю сам в `KNOWLEDGE_GAPS.md` без явного решения,
   учитывая явную оговорку источника о невалидированности.
3. **"Свип"-свеча** — свеча с длинной тенью, снимающая ликвидность — по сути
   то же самое, что уже реализованный `detect_sweep()`/SFP, другое название
   для уже известного примитива.

**Модуль бота**: `ta_extra.find_sr_zones`/order block (поглощение, есть),
`detect_sweep` (свип-свеча, есть). "Рельсы" — единственный потенциально новый,
но НЕвалидированный самим источником примитив — не рекомендую добавлять в код
без отдельного бэктеста, инструктор сам предупреждает о ненадёжности как
самостоятельного сигнала.
