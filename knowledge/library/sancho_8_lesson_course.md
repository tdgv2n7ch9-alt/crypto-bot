# Курс инструктора "Санчо" (Инструктор B), 8 занятий + короткие клипы по R:R/БУ

**Полностью дублирует METHODOLOGY_CORE.md** — этот курс уже прочитан целиком и
детально извлечён в отдельные разделы METHODOLOGY_CORE.md в прошлых сессиях
(построчно, с таймкодами). Данный файл — компактный конспект-указатель для
раздела 🎥 КОНСПЕКТЫ (Этап 3), не повторная экстракция.

**Урок 1. Структура** (→ §1 METHODOLOGY_CORE.md):
- Свинг хай/лоу — 5-свечной фрактал (не 3-свечной), 3-свечные свинги —
  структура младшего ТФ, не ошибка. Бот уже использует 5-барный фрактал
  (`FRACTAL_SIDE_BARS=2`) — соответствие подтверждено.
- Слом структуры ТЕНЬЮ свечи (без закрытия тела) — валидный слом по этому
  инструктору. Бот сверяет по теням (`high`/`low`), не по телу — тоже
  соответствие.
- 3-уровневая иерархия: свинг-структура (тренд) → суб-структура (коррекция
  против тренда) → майнер-структура (начинается, когда суб-структура ломается
  — сигнал вероятного конца коррекции). Бот НЕ реализует эту иерархию (только
  бинарный BOS/CHoCH) — честный пробел.
- Мультитаймфрейм-синхронизация: старшие ТФ (месяц/неделя/день) приоритетнее
  младших; таблица "что происходит на младших ТФ при коррекции на старшем".
- Fibonacci: сетка по телам свечей точнее, чем по теням (тела не варьируются
  между брокерами). 50% = граница premium/discount, не точка входа сама по
  себе.
- Риск-менеджмент: 1-2% на сделку, ~1500 подряд убыточных сделок с риском 1%
  чтобы потерять весь депозит, ~70 — чтобы потерять половину.

**Урок 2. Ликвидность и имбаланс** (→ §4): глубина входа внутри FVG даёт разный
R:R — вход на 0% заполнения ≈ 1:4.5, на 50% ≈ 1:4, на 100% ≈ 1:3.5 (обратная
зависимость: глубже вход = выше вероятность реакции, но ниже R:R). Бот не
различает точку входа внутри FVG по глубине — упрощение.

**Урок 3. OB, MB, BB, Wick** (→ §18.1): критерий валидности OB в живом коде
(`bot.detect_order_blocks()`) не проверяет снятие ликвидности предыдущего
свинга — только относительный размер тела. Отдельно подтверждено: OB должен
строиться по ТЕЛУ свечи (не хай/лоу) — расхождение с частью боевого кода
(детали и статус фикса — см. §18.1, patch 07 order-block уже в shadow-режиме).

**Урок 4. Дивергенция, TDP, TTS** (→ §10, частично): TDP/TTS — акронимы НЕ
расшифрованы источником, честно зафиксировано как "н/д", не расшифровка "на
глаз".

**Урок 5. Время и киллзоны** (→ §7-8): PO3 фрактален (не только на дневном
ТФ), killzone-часы с явной поправкой к докстрингу `get_killzone_status()` —
NY Open в коде 14:00-16:00 (2ч), в уроке явно 14:00-17:00 (3ч) — код обрезает
последний час; London Close (17:00-19:00) в коде помечен "не подтверждено
источником", но этот источник + независимо `37_Стрим_Логика_запуска_
алгоритма.md` явно называют именно это окно — потенциально закрывает TODO.

**Урок 6. Рендж и Вайкофф** (→ §6): независимая перепроверка Accumulation/
Distribution с явным акцентом на контекст ("схемы Вайкоффа без контекста
провоцируют открывать позиции в неверном направлении"). R:R у этого автора на
практике ~1:4 ("у меня такие сделки обычно получаются с RR 1х4 или выше").

**Урок 7. Сетапы / Урок 8. Стили торговли** (→ §12, §9): мультитаймфрейм-
workflow, стили торговли по таймфреймам (свинг — D1/4h/1h, скальп — младшие
ТФ), DXY/S&P500 как конфлюенс-фактор (DXY — отрицательная корреляция),
funding rate как доп. фактор перекупленности. **Скальп-сделки: explicit
минимум R:R 1:4** — "низкий винрейт, но всё равно оставаться прибыльным".

**Вход в сделку, БУ, RR 1/2/3** (→ §9, короткие клипы того же/связанного
трека): риск на сделку 0.5-2% (чаще 0.5%, "2% уже много"); **explicit минимум
R:R 1:15** для входа в сделку ("минимум от 1 к 15" — если расчётный R:R
меньше, автор улучшает вход, а не снижает порог); БУ (безубыток) переносится
ТОЛЬКО после подтверждения структуры (пробит структурный хай/лоу в сторону
сделки) — explicit запрет переносить SL в BU из страха стопа.

**Синтез по R:R после всех источников курса**: диапазон явных минимумов
1:3 (нижняя граница, методичка + Санчо) — 1:15 (верхняя граница, "Вход в
сделку RR 1"). Бот на гейте 1.5 — близко к нижней границе диапазона, не
радикально мягче всех источников. Подробный разбор неоднозначности
транскрипции ("1,4 либо 1,5" — R:R 1.4-1.5 или 1:4-1:5, ASR не даёт
однозначного прочтения) — см. METHODOLOGY_CORE.md §9 целиком.
