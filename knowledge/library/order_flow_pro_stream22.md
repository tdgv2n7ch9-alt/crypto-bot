# 22. Стрим Торговые инструменты (Pro), день 1 (поток 7.0) — второе подтверждение IOF

**Прочитано выборочно (~127 мин, интро + разделы STB/BTS и Order Flow),
Pro-модуль курса, но содержание базовое по сути**: STB/BTS (sell-to-buy/
buy-to-sell) — та же манипуляция-снятие-ликвидности-перед-order-block'ом,
что уже многократно задокументирована (просто с явным акцентом на то, что
снятие ликвидности ОБЯЗАТЕЛЬНО предшествует манипуляции — без этого STB/BTS
не бывает).

**"Order Flow" в этом уроке = ДРУГОЙ смысл, чем в батче 15 ("Кластерный
анализ")**: там order flow — tick-level дельта покупок/продаж (данные,
которых у бота нет). Здесь — "order flow" означает УСТОЙЧИВЫЙ
институциональный bias, подтверждённый ПОВТОРЯЮЩИМИСЯ циклами захват-
ликвидности→реакция, в отличие от "просто тренда" (голое обновление хай/лоу
без явного захвата ликвидности). **Это ВТОРОЕ независимое подтверждение**
уже задокументированного `KNOWLEDGE_GAPS.md` п.19 (Institutional Order Flow
как более общая концепция, чем BOS/CHoCH — order flow может сохраняться,
даже когда локальная структура формально "сломана") — не новый пункт,
усиливает вес.

**Дополнительный нюанс, реально новый для документа**: явно сказано, что
BOS в контексте order flow "используется чисто как модель для входа, это
НЕ 100% слом структуры" — то есть BOS-триггер входа и "настоящий" структурный
разворот здесь СОЗНАТЕЛЬНО разделяются этим инструктором — согласуется (не
дублирует дословно, но той же природы) с уже известным пробелом "локальная
vs глобальная структура" (батчи 11, 17).

**Приоритет стека конфлюенса** (личное правило инструктора, не абсолютный
канон): 1) имбаланс (перекрытие) — важнее всего, 2) order block, 3) STB/BTS
на LTF внутри order block'а. Минорная деталь, не gap — скорее персональная
эвристика приоритизации факторов, которые бот и так все учитывает по
отдельности (без явного порядка важности).

**Модуль бота**: нет нового кода-уровня — те же примитивы (sweep/BOS/order
block/imbalance), что уже реализованы и cross-referenced в других
конспектах. Продолжение (день 2, "23. Стрим Практический модуль") -- не
входит в этот батч, отдельный файл в очереди.
