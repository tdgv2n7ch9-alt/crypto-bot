# 16. Фундаментальный анализ (поток 7.0) — честно выборочно, не построчно

**126 минут, прочитано выборочно (~30%, интро + ключевые повторяющиеся темы), не
целиком.** Причина: инструктор САМ явно оговаривает в начале — "фундаментальный
анализ больше необходим для среднесрочного и долгосрочного спекулирования" —
это методология для ПОЗИЦИОННОЙ/инвестиционной торговли (недели-месяцы), largely
ортогональная тому, что делает бот (краткосрочные технические сигналы с
конкретным R:R/SL/TP на 1-24ч горизонте). Основная часть содержания (разбор
whitepaper, оценка команды проекта, поиск нарратив-трендов вроде NFT/Metaverse/
GameFi/M2E для среднесрочного холда) — методология due-diligence для ЧЕЛОВЕКА,
выбирающего, ВО ЧТО инвестировать на недели/месяцы, не про то, КОГДА войти
технически — не переводится в код бота почти никак.

**Реально пересекается с ботом**:

1. **BTC/ETH-доминация и общая капитализация как индикатор alt-сезона** —
   источник: падение доминации BTC = деньги перетекают в альты (alt rotation),
   рост доминации = деньги возвращаются в BTC. Бот УЖЕ считает `btc_dominance`/
   `eth_dominance`/`total_mcap` (`bot.py` ~1359-1389, ~8453-8497) — но
   **используется ТОЛЬКО для отображения в текстовом обзоре рынка**, НЕ как
   фактор скоринга/гейта для отдельных альткоин-сигналов (не boost/penalty на
   основе тренда доминации). Конкретный, проверяемый по коду **GAP-кандидат**
   (не добавляю в `KNOWLEDGE_GAPS.md` явным пунктом — это скорее "фича не
   реализована", чем "методология расходится с кодом", разные категории, но
   стоит зафиксировать здесь).
2. **Новости/волатильность на среднем ТФ** — сильное новостное событие (пример
   инструктора: Ethereum Merge) может пробить структуру даже против
   преобладающего тренда, но это отыгрывается техническим анализом на СТАРШЕМ
   ТФ (4h+), не младшем ("не лезьте на 15 минут при поиске разворота
   среднесрочной позиции") — согласуется с уже известным принципом бота
   "старший ТФ приоритетнее" (см. `METHODOLOGY_CORE.md` пп. про приоритет HTF),
   не новое.
3. **Общая капитализация как самостоятельный "актив" с зонами S/R** — источник:
   агрегированный total market cap chart ведёт себя предсказуемо (уровни,
   реакции), в основном из-за доминирования BTC/топ-10 в составе индекса — не
   реализовано ботом (нет отдельного анализа TOTAL/TOTAL2 капитализации как
   графика), но и не входит явно в скоуп краткосрочных сигналов бота — скорее
   макро-контекст, не gap первого порядка.

**Осознанно НЕ дочитано подробно** (за пределами выборки): whitepaper-анализ,
оценка команды/инвесторов проекта, детальный разбор ICO/токеномики конкретных
исторических нарративов (NFT/Metaverse/GameFi/M2E) — методология due-diligence
для человека-инвестора, не для бота. Если владелец захочет отдельно проработать
эту часть (например для раздела "Предложения из библиотеки" — образовательный
контент подписчикам про макро-контекст) — готов вернуться и дочитать подробнее
по отдельному запросу.

**Модуль бота**: `bot.py` (`btc_dominance`/`eth_dominance`/`total_mcap` — есть,
но display-only, см. п.1 выше), `onchain_metrics.py` (`fetch_usdt_dominance` —
похожий, уже реализованный концепт для другого индекса).
