# Курс C, Урок 2 — Продолжение

**Философское отступление (не новое)**: критика ICT-хайпа вокруг "новых"
терминов для классических инструментов (например "поглощение" ~150-200 лет
как концепт) — тот же тезис, что уже звучал у других инструкторов этого
батча ("названия придумывают, суть одна").

**Конкретная, количественно заданная деталь — "4 касания" как критерий
подтверждённой структуры**: источник явно определяет: 2 максимума + 2
минимума = "полноценная структура" на данном таймфрейме (4 touch-points).
"Синхронизация" двух периодов означает требование, чтобы МЛАДШИЙ таймфрейм
показал ИДЕНТИЧНУЮ по направлению и форме структуру (те же 4 касания) —
только тогда вход с LTF считается надёжно подтверждённым HTF-контекстом.
Точка входа (где именно на LTF) определяет, за что ставится SL (за
локальный слом LTF, не за весь HTF-диапазон).

**Сверка с ботом**: `ta_extra.swing_points()` детектирует фрактальные
свинги без явного порога "минимум 4 касания = подтверждённая структура" —
структура у бота либо есть, либо нет (бинарно), нет отдельного счётчика
"насколько подтверждена". Небольшой, но конкретный, количественно заданный
нюанс — не завожу отдельным пунктом в `KNOWLEDGE_GAPS.md` без более
широкой проверки (может оказаться, что 2 hi + 2 lo это просто описание
МИНИМАЛЬНОГО необходимого числа точек для swing_points() и так уже
неявно требует, не пробел, а просто другая формулировка того же).

**Модуль бота**: `ta_extra.swing_points()` — уже реализует базовую
фрактальную детекцию, явного счётчика "4 касания" нет.
