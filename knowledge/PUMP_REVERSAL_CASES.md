### Структура — честная проверка на CHoCH (Bybit klines, 5м/15м/1ч)
- **1H**: непрерывная серия HH/HL с 06:00 до 14:00 UTC (0.0003488 → хай 0.0009222 в 14:00-часовой
  свече). Текущая (ещё формирующаяся) 15:00-часовая свеча: open 0.000846, high 0.0008773,
  **low 0.000615**, close(так далеко) 0.0007671 — длинный верхний хвост + просадка, но **закрытие
  ещё выше предыдущего 1H-лоя (0.0006969)** — структура 1H **пока не сломана**.
- **15м**: свеча 15:00–15:15 пробила локальный минор-лоу (0.000615 против 0.0007377/0.0008021/
  0.0008107 на предыдущих 15м-свечах) — технически снятие структурного лоя произошло, НО свеча
  закрылась в середине диапазона (0.7157 из 0.615–0.8773), а не у минимума — похоже на
  стоп-хант/снятие ликвидности, а не подтверждённый слом с закреплением. Следующая 15м-свеча
  (15:15) попыталась вернуться выше (хай 0.7824), не переписав хай пампа.
- **fa_engine.build_full_analysis('AKE') → block3_smc**: `"type": null, "label": "структура не
  определена", "aligned": null` — движок САМ не увидел подтверждённого CHoCH/BOS. block11_trade_plan
  всё равно построил механическую SHORT-сделку от обычных S/R-зон (bias=SHORT от дневного
  LH/LL) — это generic zone-математика, НЕ равнозначна подтверждённой реверсивной структуре, и
  её нельзя было выдавать владельцу как "сетап готов".

### Вердикт, выданный владельцу
Честно: **сетапа на слом структуры пока нет** — есть явный перегрев (памп +282%/24ч, толпа в
шортах 70-76% по Bybit account-ratio, vol/mcap 211%, резкий wick-sweep на 15м), но ни 15м, ни 1H
не дали закрытия ниже подтверждённого структурного лоя с ретестом. Условие входа сформулировано
как CHoCH на 15м-1H (закрытие ниже локального лоя серии HH/HL) + ретест сломанного уровня снизу.
Без выдуманных числовых entry/SL/TP уровней — ждём подтверждения.

### Почему это эталонный кейс для Pump→Reversal / казино-флага (#214)
Иллюстрирует ровно сценарий, под который задача #214 проектируется: экстремальный
vol/mcap (211-221%, порог формулы >50%), однобокое позиционирование по account-ratio, и при
этом честный движок (`block3_smc`) НЕ подтверждает структуру автоматически — то есть даже с
казино-флагом (снижающим Rocket Score) для этого класса активов ключевая защита — отсутствие
автоматической генерации entry/SL/TP без подтверждённого CHoCH, что уже так в
`build_trade_from_structure`, но `block11_trade_plan`/`block5_checklist.trade` строят
"сделку" от простых S/R-зон независимо от `block3_smc` — это несогласованность движка (не баг
в смысле неверных чисел, но риск того, что карточка выглядит как "сетап готов", когда реально
подтверждённой реверсивной структуры нет). Стоит учесть при следующей ревизии verdict-текста
(`block13_verdict`) — возможно требуется явное предупреждение "SMC-структура не подтверждена"
рядом с любым trade_plan, построенным только от S/R-зон без CHoCH. Не меняю это сейчас
(боевая логика/пороги — только с явного "да" владельца) — фиксирую находку для решения.

**Полный референс-профиль "монеты под управлением" для порогов радара (П-Памп v3)**:

2025-09-27, см. выше). Это сочетание -- предлагаемый эталонный сигнальный профиль для
калибровки порогов Фазы 1/2/3 радара (задачи #220-225).

**Программная проверка доступности holder-данных (2026-07-15, наш ETHERSCAN_API_KEY,
живой тест через Etherscan V2 unified API, `chainid=56` для BSC)** — ЗАБЛОКИРОВАНО
честно: ВСЕ 4 протестированных эндпоинта (`tokenholderlist`, `tokeninfo`,
`tokenholdercount`, и даже `tokentx` — обычно бесплатный на других сетях) вернули
одинаковую ошибку: `"Free API access is not supported for this chain. Please upgrade
your api plan for full chain coverage."` — наш текущий ключ на бесплатном тарифе НЕ
покрывает BSC вообще, ни для одного типа запроса, включая переводы токена. Это
означает:
- Holder-слой радара (порог топ-10 >60%/>80%, автоматический 🎰-флаг) — не может быть
  реализован программно на текущем тарифе.
- Мониторинг оттока топ-кошельков на биржи (Фаза РАЗВОРОТ, строка "📤 Топ-кошельки
  двигают на биржи") — тоже заблокирован тем же ограничением.
- Персональный алерт владельца по 3 конкретным кошелькам AKE — технически нереализуем
  без доступа к `tokentx`/аналогу; кроме того, владелец дал адреса в УСЕЧЁННОМ виде
  (`0x2733...d1f42c` и т.п.) — для реального запроса к API нужны ПОЛНЫЕ 42-символьные
  адреса, которых у нас нет и получить программно без holder-list эндпоинта (тоже
  заблокирован) не из чего.
- **Решение нужно от владельца** (вопрос денег/API-подписок — точка владельца по
  CLAUDE.md): либо оплатить тариф Etherscan с покрытием BSC, либо найти альтернативный
  источник (например Moralis/Covalent имеют бесплatные тарифы с BSC-покрытием — НЕ
  проверено нами, нужна отдельная разведка с проверкой актуальных лимитов, прежде чем
  предлагать как факт). До решения — holder-слой в П-Памп v3 (задачи #220-225) остаётся
  "н/д на BSC", как и было изначально предусмотрено пунктом 6 задания владельца.
