# Патч 1: Killzone-часы (`bot.py:get_killzone_status`) — 🔴 высокий приоритет

## Что меняет

- Часы сессий приводятся в соответствие с `knowledge/METHODOLOGY_CORE.md §8`
  (источник — `Урок 5. Время и киллзоны.mp4` [1662s-1902s]):
  - Азия: `01:00-04:00` → `до 08:00`
  - Лондон: `10:00-12:00` → `09:00-12:00`
  - Нью-Йорк: `16:00-18:00` → `14:00-16:00` (расхождение было 2 часа)
  - "London Close"/"NY Close" зоны оставлены как есть — источник их не упоминает, не
    подтверждены и не опровергнуты, вне скоупа этого патча.
- Попутно восстановлен читаемый Cyrillic-текст (названия зон, докстринг) — был стёрт ещё
  в древнем коммите `62080d6` ("v45 restore v43"), логика при этом не страдала, только
  подписи были невидимы (пустые строки вместо текста).

## Почему это не «добавление фичи»

Это единственный патч из пяти, который правит **уже активно используемый** код, а не
добавляет новый. `get_killzone_status()` читается в `fa_engine.py:190-191` (чек-лист
block5, +8 баллов "killzone активна") и `fa_engine.py:258-259` (текст сценария сделки).
Неверные часы означают, что этот фактор мог засчитываться (или нет) в неправильное время
суток на каждом прогоне анализа, с момента коммита `62080d6`.

## Ожидаемый эффект

Не измерен. Гипотеза: часть исторических сигналов, где "killzone активна" засчитывался
как фактор в неверное окно времени (16:00-18:00 вместо реального 14:00-16:00 по NY),
получала на +8 баллов больше/меньше rocket-score, чем должна была бы по методике курса.
Прямой эффект на win-rate не оценивался.

## Как измерим на журнале

История уже закрытых сигналов размечена СТАРЫМИ часами — ретроспективно пересчитать
нельзя (в `signal_journal` не сохраняется отдельно, была ли killzone активна на момент
сигнала, только итоговый rocket-score). После применения патча: сравнивать распределение
`grade`/`rocket_score` по новым часам за несколько недель с уже накопленной историей —
не строгий A/B-тест, но единственный практически доступный способ без ретроактивной
разметки.

## Риск применения

Низкий. 5 чисел в одной функции + текстовые строки (не влияющие на логику). Не трогает
торговые формулы/пороги/R:R-гейт.
