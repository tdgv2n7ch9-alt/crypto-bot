# Патч 4: RSI/индикатор-дивергенция — 🟡 средний приоритет

## Что меняет

Новая аддитивная функция `ta_extra.detect_price_indicator_divergence(candles, period)`.
**Не подключена ни к одному вызывающему коду** — само применение патча ничего не меняет
в поведении бота.

Реализует классическую и скрытую дивергенцию цена-vs-RSI из
`knowledge/METHODOLOGY_CORE.md §10` (источник: `Урок 4. Дивергенция, TDP, TTS.mp4`
[136s-367s]) — **контрарианская трактовка**, не наивная "дивергенция = сигнал входа":
- Классическая (цена HH/RSI LH или LL/HL) — источник трактует как признак ВОЗМОЖНОЙ
  коррекции против HTF-bias, не сигнал входа по направлению дивергенции.
- Скрытая (обратная классической) — источник трактует как подтверждение продолжения
  HTF-тренда.
- Бычья скрытая версия выведена по симметрии с явно описанной медвежьей — честно
  помечено в докстринге как вывод по аналогии, не прямая цитата источника.

## Отдельный, не путать с уже известным пробелом

Это НЕ SMT (кросс-активная дивергенция между разными монетами, уже отмечена в
`KNOWLEDGE_GAPS.md` п.13) — это классическая дивергенция цена/индикатор на ОДНОЙ монете.
Два разных, независимых пробела.

## Реализация

Переиспользует уже существующую чистую функцию `rsi()` (вызывает её на срезе `closes`
до индекса каждого свинга) — не дублирует формулу RSI. Сравнивает 2 последних
фрактальных свинг-хая (медвежьи варианты) и 2 последних свинг-лоя (бычьи варианты).

## Ожидаемый эффект

Не измерен.

## Как измерим на журнале (если/когда будет решение подключить)

Контрарианская трактовка сложнее для прямого A/B — потребует не простого "+X баллов"
фактора, а модификатора существующих факторов (например, "снизить уверенность в сигнале,
если видна классическая дивергенция против направления сделки"). Первый шаг измерения:
аддитивное поле в `signal_journal` (bool на каждый тип дивергенции на момент сигнала),
накопить историю, затем смотреть корреляцию с исходом — не встраивать в scoring до этого.
