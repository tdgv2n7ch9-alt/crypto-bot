# Патч 2: R:R-гейт 1.5 → 2.0 (`ta_extra.py:SR_MIN_RR_TP1`) — 🔴 высокий приоритет, ТОРГОВАЯ ЛОГИКА

## ⚠️ Это торговая логика — правило ночи требует диф, а не применение

Патч написан и протестирован (не сломал существующие тесты), **но НЕ должен
применяться без отдельного явного «да»**, независимо от остальных четырёх патчей.

## Что меняет

`SR_MIN_RR_TP1 = 1.5` → `2.0`. Единственная константа, гейт применяется во всех путях
`real_full_analysis`/`fa_engine` (подтверждено расследованием бага №5 ранее в этой же
сессии — обхода нет).

## Почему именно 2.0, а не 1:3 или 1:15

`knowledge/MISMATCH_REPORT.md` п.1: 4 источника методики дают разброс R:R-минимума от
1:3 (личная методичка владельца + один из инструкторов курса, "Санчо") до 1:15 (другой
явный источник). Взять "среднее" необоснованно — источники сами расходятся в 5 раз.
**2.0 — намеренно консервативный первый шаг**: выше боевого 1.5, но заметно ниже даже
нижней границы явных рекомендаций курса (1:3) — минимальное движение в сторону
методики, не резкий скачок.

## Ожидаемый эффект

Не измерен. Гипотеза: снизит число сигналов, проходящих гейт (более строгий фильтр),
может поднять win-rate/avg-R за счёт отсева сетапов с слабым R:R — но это гипотеза, не
установленный факт.

## Как измерим на журнале — ОБЯЗАТЕЛЬНО до применения

`signal_journal` уже хранит поле `rr`/`rr_tp1` на каждой записи независимо от того, что
боевой гейт сейчас 1.5. **Перед применением патча** нужен ретроспективный анализ:
разбить уже закрытые сделки (37 на момент записи) по бакетам R:R (1.5-2.0 / 2.0-3.0 /
3.0+) и сравнить win-rate/avg-R между бакетами. Если данных пока мало для значимого
вывода (индустриальный ориентир 30+ для предварительных, 100+ для надёжных, см.
`BACKTEST_REPORT.md`) — честно ждать накопления, не применять патч вслепую.

**Этот патч — гипотеза для проверки, не готовый к слепому мержу вывод.**

## Риск применения

Средний-высокий (торговая логика). Существующие тесты (`tests/test_ta_extra_structure.py`)
проверяют МЕХАНИЗМ гейта (`rr_gate_pass == rr_tp1 >= SR_MIN_RR_TP1`), не конкретное
значение порога — патч их не ломает, но это не означает, что 2.0 — правильное число,
только что код продолжает работать корректно с новым числом.
