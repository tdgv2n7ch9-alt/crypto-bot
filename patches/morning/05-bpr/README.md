# Патч 5: BPR (Balanced Price Range) детектор — 🟡 средний приоритет

## Что меняет

Новая аддитивная функция `ta_extra.detect_bpr_zones(candles)` + константа
`BPR_MAX_BAR_GAP = 5`. **Не подключена ни к одному вызывающему коду**.

Реализует BPR из `knowledge/METHODOLOGY_CORE.md §4` (источник: `BPR.mp4` [0s-170s]) —
область пересечения ДВУХ ПРОТИВОПОЛОЖНЫХ имбалансов (FVG), формирующихся друг за другом
при смене направления. Более узкая и точная триггерная точка, чем широкий FVG сам по
себе — цена стремится протроговать BPR на 50% либо 100% диапазона.

## Почему свой скан, а не переиспользование `find_fvg_zones()`

`find_fvg_zones()` уже существует и используется в `fa_engine.py` block4 (только для
отображения POI на карточке `/full`), но не хранит индекс формирования каждого FVG —
для поиска ПАР соседних по времени противоположных FVG индекс обязателен. `detect_bpr_
zones()` делает независимый 3-свечной скан той же формации, не меняя и не дублируя
`find_fvg_zones()` для её текущей задачи.

## Ожидаемый эффект

Не измерен.

## Как измерим на журнале (если/когда будет решение подключить)

Аддитивное поле `bpr_zone_hit: bool` в `signal_journal` (был ли вход внутри BPR-зоны на
момент сигнала), накопить историю, сравнить win-rate сигналов с BPR-confluence против
сигналов без него.

## Риск применения самого патча (добавление функции)

Минимальный — новый код, ничего не вызывает. Протестировано: находит реальное
пересечение bull/bear FVG на синтетических данных, возвращает пустой список при
отсутствии пересечений.
