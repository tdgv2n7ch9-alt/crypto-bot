# patches/morning/ — готовый пакет решений к утру (ночная сессия #3, Блок I)

Для каждого из 5 патчей ночи #2 — статус диффа + историческая доказательная база из
`PATCH_IMPACT.md` (ночь #3), чтобы решение "да/нет" принималось не вслепую по
качественному описанию, а с числами на реальной истории (2864 боевых / 1823
патченых сделок, 100 символов, ~12 месяцев). **Ничего не применено без вашего "да"
— это не автоматическое внедрение, а подготовленный пакет.**

## Патч 01 (killzone-hours) + Патч 02 (RR-гейт 1.5→2.0)

**Диффы актуальны, применяются чисто** (`git apply --check` подтверждён на текущем
`main`) — `01-killzone-hours/bot.py.diff`, `02-rr-gate/ta_extra.py.diff`.

**Историческая доказательная база (`PATCH_IMPACT.md`)**: оба патча прогонялись
СОВМЕСТНО (эффект по отдельности не разделён, честно помечено как ограничение) —
**совместно они УХУДШИЛИ метрики**: win rate 53.6%→43.6% (-10pp), средний R
+0.832→+0.744, expectancy +0.815→+0.720, profit factor 2.86→2.37. Объём сделок упал
на 36% (2864→1823).

**Рекомендация (не решение — это ваше)**: контринтуитивный результат — не спешить с
"да" на оба сразу. Разумный следующий шаг перед решением: ещё один исторический
прогон с патчами ПО ОТДЕЛЬНОСТИ (01 без 02, 02 без 01), чтобы понять, тянет ли вниз
именно ужесточение R:R-гейта, именно смена killzone-часов, или оба вместе создают
эффект, которого нет у каждого поодиночке. Это не сделано в рамках бюджета ночи #3.

## Патч 03 (Breaker/Mitigation), 04 (RSI-дивергенция), 05 (BPR)

**Диффы БОЛЬШЕ НЕ ПРИМЕНЯЮТСЯ как есть** — честно, найдено при подготовке этого
пакета: функции `ta_extra.classify_breaker_or_mitigation()`,
`ta_extra.detect_price_indicator_divergence()`, `ta_extra.detect_bpr_zones()` уже
ЖИВЫЕ в `ta_extra.py` (добавлены ночью #2, Блок 1, для `shadow_engine.py`) — `git
apply --check` на старые `.diff` из этой ночи проваливается («patch does not
apply», функции уже существуют). Это НЕ откат/потеря патчей — сами функции работают
и накапливают данные через shadow-контур прямо сейчас. Просто "диффы добавления
функции" больше не нужны — единственное, чего не хватает, это **диффа подключения**
(вызов этих функций как реального фактора/гейта внутри `fa_engine.py`/
`real_full_analysis()`), который здесь НЕ подготовлен — это уже изменение боевой
сигнальной логики, требует отдельного явного решения, не просто "применить диф".

**Историческая доказательная база (`PATCH_IMPACT.md`, информационные теги на
patched-сделках, не гейтующие)**:
- **03 Breaker/Mitigation**: breaker (n=134) средний R +1.128, mitigation (n=1459)
  +0.704 — заметная разница, breaker выглядит сильнее, но breaker — маленькая доля
  выборки (~7%).
- **04 RSI-дивергенция**: сделки с классической дивергенцией ПРОТИВ направления
  (n=257) дали средний R всего +0.292, против +0.818 без такой дивергенции (n=1566)
  — **самый убедительный сигнал из трёх**, контрарианская трактовка методики
  (§10 `METHODOLOGY_CORE.md`) получает первое количественное подтверждение и на
  живом shadow, и теперь на историческом прогоне.
- **05 BPR**: маленькая выборка (n=41 с confluence), слабо позитивное направление
  (+0.776 vs +0.743) — недостаточно данных для решения.

**Рекомендация (не решение)**: патч 04 — самый обоснованный кандидат на "следующий
шаг" (не гейт, а МОДИФИКАТОР уверенности — например, понижать грейд сигнала при
дивергенции против направления, не блокировать полностью) — уже предложено так в
`patches/04-rsi-divergence/README.md` изначально. 03/05 — продолжать копить данные
(живой shadow-контур + возможные будущие исторические прогоны), решения пока рано.
