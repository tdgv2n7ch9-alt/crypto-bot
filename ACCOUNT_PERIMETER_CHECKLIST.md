# ACCOUNT_PERIMETER_CHECKLIST.md — периметр аккаунтов (Пакет SECURITY-HARDENING М6)

Бот не может пройти эти шаги сам (требуют входа в аккаунты владельца) — пошаговый
чек-лист, рассчитан на прохождение с телефона. Отмечайте `[x]` по мере прохождения,
это ваш собственный список (Claude Code не может подтвердить выполнение — только
владелец видит свои настройки аккаунтов).

## 1. GitHub — 2FA

- [ ] Открыть github.com → фото профиля (правый верхний угол) → **Settings**
- [ ] **Password and authentication** → **Two-factor authentication** → **Enable**
- [ ] Выбрать приложение-аутентификатор (Google Authenticator / Authy / встроенный
      в менеджер паролей) — НЕ только SMS (SMS можно перехватить через подмену SIM)
- [ ] Сохранить recovery codes в надёжном месте (менеджер паролей, НЕ скриншот в
      Telegram/почте) — без них можно потерять доступ к аккаунту при смене телефона

## 2. GitHub — branch protection на `main`

- [ ] Открыть репозиторий `tdgv2n7ch9-alt/crypto-bot` → **Settings** → **Branches**
- [ ] **Add branch protection rule** (или Add rule) → Branch name pattern: `main`
- [ ] Включить **Require a pull request before merging** — ИЛИ, если это помешает
      боевому рабочему процессу (бот и Claude Code коммитят напрямую в `main` через
      сессии), как минимум включить **Do not allow force pushes** — это ключевая
      защита от переписывания истории при угоне аккаунта, не мешает обычным коммитам
- [ ] Проверить, что **Allow force pushes** выключен (по умолчанию должен быть)
- [ ] Опционально: **Require signed commits**, если готовы настроить GPG/SSH-подпись
      (не обязательно для этого чек-листа, но повышает доверие к истории)

## 3. GitHub — Personal Access Token, используемый ботом

- [ ] **Settings** → **Developer settings** → **Personal access tokens** → найти
      токен, которым пользуется `GITHUB_TOKEN` в Railway Variables
- [ ] Проверить **scope** — должен быть минимальным (доступ к репозиторию, не
      `admin:org`/`delete_repo` и т.п., если сейчас шире необходимого — сузить)
- [ ] Проверить **expiration** — если "No expiration", рассмотреть замену на токен
      с сроком действия (например, 90 дней) и напоминанием себе на обновление

## 4. Railway — 2FA

- [ ] Открыть railway.app → аватар (правый верхний угол) → **Account Settings**
- [ ] Найти раздел двухфакторной аутентификации (обычно **Security** /
      **Two-Factor Auth**) → включить через приложение-аутентификатор
- [ ] Сохранить recovery codes так же, как для GitHub (см. п.1)

## 5. Railway — session review

- [ ] В том же **Account Settings** найти список активных сессий/устройств
      (если Railway это показывает в UI на момент прохождения чек-листа)
- [ ] Разлогинить любые незнакомые сессии/устройства
- [ ] Проверить **Team/Project members** проекта `resilient-respect` — только
      владелец должен иметь доступ, никаких лишних приглашённых аккаунтов

## 6. Telegram — двухэтапная аутентификация + скрытие номера

- [ ] Telegram-приложение → **Settings** → **Privacy and Security**
- [ ] **Two-Step Verification** → включить, задать облачный пароль (это ОТДЕЛЬНО от
      SMS-кода при входе — защищает от угона номера/SIM-swap)
- [ ] Указать резервный email на случай, если забудете облачный пароль
- [ ] **Privacy and Security** → **Phone Number** → выставить **Who can see my
      phone number** на **Nobody** (или **My Contacts**, если нужно, чтобы видели
      знакомые) — скрывает номер от посторонних, кто найдёт бота/владельца в поиске
- [ ] Проверить **Active Sessions** (там же, в Privacy and Security) — разлогинить
      незнакомые устройства, если есть

## 7. BOT_TOKEN — где хранится (проверка, не действие)

- [ ] Подтвердить, что `BOT_TOKEN` присутствует ТОЛЬКО в Railway Variables
      (сервис crypto-bot), не в коде — это уже подтверждено в `SECRETS_AUDIT.md`
      (Пакет SECURITY-HARDENING М2, gitleaks-скан истории чистый), но стоит
      перепроверить глазами при следующем визите в Railway Variables, что ничего
      нового не появилось в коде за это время

## Честно про этот чек-лист

- Ни один пункт здесь бот не может выполнить или подтвердить сам — все требуют
  входа в личные аккаунты владельца. Claude Code предлагает список, владелец
  отмечает и решает.
- Прохождение п.1-6 в среднем 10-15 минут с телефона (по спецификации владельца
  "владелец пройдёт с телефона").
- См. также `RUNBOOK_SECURITY.md` (Пакет SECURITY-HARDENING М7) — что делать,
  если один из этих периметров УЖЕ нарушен (не для профилактики, а для реакции
  на инцидент).
